# Инструкция: Создание MVP

Ты создаёшь работающий MVP с живой базой данных. Это продукт, который можно показывать реальным пользователям.

## Входные данные

- PRD-файл — там описан продукт и его функции
- `landing-design.md` — инструкции по дизайну

## Главные принципы

### Что делаем

- Работающий продукт с живой базой данных
- CRUD-операции (создание, чтение, обновление, удаление)
- Красивый UI по инструкциям из `landing-design.md`
- Seed data — база сразу заполнена тестовыми данными
- Готов к деплою на Vercel

### Чего НЕ делаем

- Никакой авторизации
- Минимум зависимостей

## Технический стек

**Next.js (App Router)** + **Tailwind CSS** + **Neon** (PostgreSQL)

## Создание проекта

```bash
npx create-next-app@latest mvp --typescript --tailwind --eslint --app --src-dir --no-import-alias --yes
cd mvp
npm install @neondatabase/serverless
```

## База данных (Neon)

**DATABASE_URL** уже есть в `.env` файле.

1. Создай таблицы SQL-запросами на основе PRD
2. Заполни тестовыми данными (seed) — чтобы UI сразу показывал контент

## Дизайн

Следуй инструкциям из `landing-design.md`:
- Выбери смелое эстетическое направление
- Уникальная типографика (не Inter, не Arial)
- Анимации и микро-интеракции
- Никакого generic AI-дизайна

## Чек-лист готовности

- [ ] `npm run dev` запускается без ошибок
- [ ] Данные сохраняются в Neon
- [ ] CRUD работает
- [ ] UI выглядит уникально и красиво
- [ ] `npm run build` проходит успешно

## Деплой

```bash
npx vercel
```

При деплое добавь переменную окружения `DATABASE_URL` в Vercel.

## Результат

Создай папку `mvp/` с Next.js проектом. После создания:

1. Запусти `cd mvp && npm run dev`
2. Скажи: "MVP готов и запущен на http://localhost:3000"
